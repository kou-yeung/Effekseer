Basic settings

Overview

You can set the parameters that are always used, regardless of the name of the node, the number of generations, and other particle settings.

Parameter

Visibility

Whether or not particles generated from this node are drawn on the screen. If you check this, particles will be drawn on the screen. When outputting for games, particles are not drawn unless this is checked.

Name

Set the name displayed in the node tree. It does not affect the operation of the effect. Set it for organizing nodes when more nodes become available.

Spawn Count

Sets the maximum number of particles generated from this node. You can also choose infinite to generate particles indefinitely until you can not generate particles with the setting of the conditions described later. However, when playing an effect from another application that incorporates Effekseer, please note that it will not disappear unless the function is called and the user deletes the effect.

Inherit Position

Sets how the position of the parent particle affects the particles generated from this node. Please refer to the following information regarding parameter contents.

Inherit Rotation

Sets how the rotation of parent particles affects particles generated from this node. Please refer to the following information regarding parameter contents.

Inherit Scale

Sets how the enlargement of parent particles affects particles generated from this node. Please see the following information regarding the contents of parameters.

Inherit **

Always

It is always affected. When the value of the parent particle is changed, the child particles are also immediately affected. When the parent particle disappears, the value at the time of disappearance is used without being changed even after it disappears.

Only on create

Use the value when the child particles were generated.

Never

It will not be affected. Whatever the value of the parent particle is, the behavior of the particle is determined only by the parameters of the child node.

Destroy after time

When survival time passes, particles are deleted. If you do not check this, particles will not be deleted after the survival time.

Destroy with parent

When the parent particle is deleted, it also deletes the child particles.

Destroy when no more children

If particles of the particle are not present and it is unlikely that children will be generated in the future, delete the particles.

Time to live

Sets the number of frames in which particles are present. If deletion is not checked due to lifetime, this parameter does not affect particle behavior.

Spawn Rate

Sets the cycle at which particles are generated. The number of frames specified will be required from generation of particles to generation of the next particle. You can specify less than 1 for this generation time, and you can generate multiple particles per frame.

Initial Delay

Specify the time from generation of parent particles to generation of child particles for the first time.
